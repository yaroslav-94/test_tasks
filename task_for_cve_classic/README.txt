Тестовое задание

Необходимо реализовать метод, который будет принимать пути до двух изображений и выдавать вероятность, с которой два
этих изображения можно считать идентичными

Данные: https://drive.google.com/drive/folders/1eZ2xa3HpxrQAbbiwn-0QqYa4xJWTgJ9Z?usp=sharing
Описание данных:
- выборка состоит из пар изображений на белом фоне
- пары можно идентифицировать по коду в именах изображений, который уникален для каждой пары

Ожидание от решения:
- исполняемый скрипт на python или jupyter notebook
- желательно оформить решение в виде класса или функции
- нужно объяснить выбор алгоритма для решения
- при необходимости нужно предоставить веса для модели
- целевые метрики: accuracy, precision, recall

На что будет обращено внимание при проверке:
1) Читаемость и воспроизводимость кода, наличие комментариев
2) Выбор и реализация метода, решающего задачу
3) Качество метода, отсутствие переобучения

--------------------------------------------------------------------------------------

Что было сделано:
1) Было предложено 2 решения:
    первое - расчет метрики SSIM для входящих изображений;
    второе - расчет метрики SSIM для тренировочного набора изображений; обучение логистической регрессии на
             тренировочном наборе; предсказание сходства изображений на основе обученной модельки
2) Был обнаружен бажик, связанный с преобразованием картинок к одному формату
3) Построены графики зависимостей метрик от порогового значения для предложенных решений

Результаты: второе решение оказалось более качественным
